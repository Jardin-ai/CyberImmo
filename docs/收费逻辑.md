# CyberImmo 商业化架构与计费系统实现蓝图 (Commercial Architecture & Billing Implementation Blueprint)

**目标 (Objective):** 指导 AI Agent (Claude Code / Cursor / Gemini CLI) 在现有 Next.js + Supabase 架构上，实现多层级订阅体系与精细化代币 (Tokenomics) 扣费逻辑。
**基准时间 (Baseline Time):** 2026-04-08

---

## 1. 核心定义与代币经济学 (Core Definitions & Tokenomics)

- **基础代币 (Base Currency):** 回声 (**Echoes**)。
- **汇率锚定 (Peg Rate):** 1 RMB = 100 Echoes。
- **消费动作矩阵 (Consumption Matrix):**
  - `TEXT_DEEPSEEK`: **5 Echoes/次** (主要用于免费用户基础消耗)。
  - `TEXT_GLM4`: **20 Echoes/次** (高情商模型，会员核心体验)。
  - `IMAGE_VISION`: **50 Echoes/张** (多模态图片理解)。
  - `VOICE_REPLY`: **100 Echoes/条** (TTS 语音合成，最高溢价资产)。

## 2. 订阅层级与配额矩阵 (Subscription Tiers & Quota Matrix)

以下为系统必须支持的五种状态映射 (Status Mapping)：

### 2.1 基础档: 寄托体验 (Remembrance Trial)

- **标识 (ID):** `FREE`
- **定价 (Price):** **￥0**
- **周期发放 (Recurring Grant):** 无。依靠每日签到获取 **20 Echoes**。
- **硬性限制 (Hard Limit - FUP):** 每日仅限 **5 次** DeepSeek 文字对话。禁止使用语音/图片功能（除非账户内拥有通过签到累积的 Echoes）。

### 2.2 基础月卡: 灵犀通讯 (Telepathy Link)

- **标识 (ID):** `MONTHLY_BASIC`
- **定价 (Price):** **￥29.9 / 月**
- **周期发放 (Recurring Grant):** 每月充入 **6,000 Echoes**。
- **特权 (Privilege):** 文本交互免扣费（白名单放行），仅对图片和语音执行扣费。

### 2.3 高阶月卡: 深度共鸣 (Resonance Plus)

- **标识 (ID):** `MONTHLY_PRO`
- **定价 (Price):** **￥49.9 / 月**
- **周期发放 (Recurring Grant):** 每月充入 **15,000 Echoes**。
- **特权 (Privilege):** 文本交互免扣费。高并发优先处理队列。

### 2.4 年度长包: 岁月长卷 (Chronicle Annual)

- **标识 (ID):** `ANNUAL_BASIC`
- **定价 (Price):** **￥268 / 年**
- **周期发放 (Recurring Grant):** 每月充入 **8,000 Echoes**。
- **特权 (Privilege):** 文本交互免扣费。

### 2.5 五年长约: 时空契约 (Spacetime Covenant)

- **标识 (ID):** `5YEAR_ULTRA`
- **定价 (Price):** **￥999 / 5年**
- **周期发放 (Recurring Grant):** 每月充入 **20,000 Echoes**。
- **特权 (Privilege):** 文本交互免扣费。解锁全套专属 UI 与优先体验权。

## 3. 数据库架构迁移指令 (Database Schema Migration Instructions)

**Agent 必须生成并执行以下 Supabase SQL (migration_003_billing.sql):**

1.  **扩展 `public.users` 表 (Extend Users Table):**
    - 新增 `subscription_tier` (VARCHAR, 默认为 'FREE')。
    - 新增 `echo_balance` (INT, 默认为 0)。
    - 新增 `subscription_expires_at` (TIMESTAMPTZ, 允许 NULL)。
    - 新增 `daily_free_text_used` (INT, 默认为 0)。
    - 新增 `last_checkin_date` (DATE, 允许 NULL)。

2.  **创建原子扣费函数 (Create Atomic RPC):**
    - 编写 RPC `process_echo_transaction(user_id UUID, action_type VARCHAR, cost INT)`。
    - **内部逻辑:** 如果 `action_type` 为 `TEXT` 且用户 tier 不为 `FREE`，直接 `RETURN TRUE` (免扣费)。
    - 否则，检查 `echo_balance >= cost`。若成立，执行扣减并插入日志至新建的 `public.billing_logs` 表；若不成立，`RAISE EXCEPTION`。

3.  **创建定时清理任务 (Create Cron Job):**
    - 使用 `pg_cron` 扩展。
    - 每天 UTC 16:00 (北京时间凌晨 0:00) 自动执行：`UPDATE public.users SET daily_free_text_used = 0;`。

## 4. 后端 API 拦截与路由网关 (Backend Interception & Routing)

**修改目标文件: `web/src/app/api/chat/route.ts`**

必须在调用 LLM 生成前，实施严格的**准入校验网关 (Access Gateway)**：

- **步骤 1: 身份与鉴权拦截 (Identity & Auth)**
  读取 `userId`，关联查询其 `subscription_tier` 与 `daily_free_text_used`。
- **步骤 2: 免费用户 FUP 熔断 (Free Tier FUP Circuit Breaker)**
  若 `tier === 'FREE'` 且 `daily_free_text_used >= 5`：
  - 拦截请求。
  - 返回 HTTP 状态码 **402 Payment Required**。
  - Payload 必须包含定制话术: `{"error": "QuotaExceeded", "message": "今天的精神波长已趋于微弱，亲人需要休息。明天再来，或者开启【灵犀通讯】以维持稳定的连接。"}`。
- **步骤 3: 多模态成本核算 (Multimodal Cost Accounting)**
  检测 `messages` 中的 `parts` 结构：
  - 若包含 `image_url`：触发 `process_echo_transaction` 扣除 **50 Echoes**。余额不足返回 **402**。
- **步骤 4: 计数器自增 (Counter Increment)**
  请求完成后，若为 `FREE` 用户，更新数据库使其 `daily_free_text_used += 1`。

## 5. 前端 UI 渲染映射逻辑 (Frontend UI Rendering Logic)

**修改目标模块: 侧边栏 (`MainSidebar.tsx`) 与 控制台 (`DashboardContainer.tsx`)**

- **UI 状态隔离 (State Isolation):**
  - 当订阅层级为 `FREE` 时：在侧边栏顶部强制显示 **“剩余免费波长: (5 - daily_free_text_used)/5”** 的红色警告进度条。
  - 当订阅层级非 `FREE` 时：隐藏波长进度条。仅柔和显示剩余 Echoes 余额。
- **签到逻辑 (Check-in Mechanism):**
  - 如果 `last_checkin_date` !== `today`，在侧边栏显示发光的 **“收集精神碎片 (签到)”** 按钮。
  - 点击触发 Server Action，`echo_balance += 20`，记录今日日期，并播放微小特效。
- **支付对比墙 (Pricing Table Component):**
  - 构建 `components/billing/PricingTable.tsx`。
  - 必须将 `MONTHLY_BASIC` (29.9元) 放置在视觉绝对 C 位 (中心突出放大)，标注 **"最受缅怀者选择 (Most Selected)"**。
  - 将 `5YEAR_ULTRA` (999元) 放置在右侧，通过颜色区隔，标注 **"数字遗产长效托管 (Long-term Custody)"**。

## 6. 开发者纪要 (Developer Notes)

- **Stripe Webhook 对接:** 后续开发 `/api/webhook/stripe/route.ts` 时，订阅成功事件必须同时更新 `subscription_tier` 和 `subscription_expires_at`，并**即刻向账户注入对应档位的首月 Echoes**。
- **无缝降级策略 (Graceful Degradation):** 若 `subscription_expires_at` 过期，系统无需删除任何用户记忆数据 (RAG 保持锁定)，仅需将 `subscription_tier` 强转为 `FREE`，并保留其剩余未消耗完的 Echoes。
